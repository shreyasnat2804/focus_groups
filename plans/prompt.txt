  ---
  CRITICAL Findings (Fix Immediately)

  1. Connection leak in api.py — conn = get_conn() is called
  before the try/finally block in create_session_endpoint,
  rerun_session_endpoint, and run_wtp_endpoint. If
  select_personas() or create_session() raises, the connection
  is never closed.
  2. No authentication on any API endpoint — Anyone who
  discovers the API URL can create sessions (triggering paid
  Claude API calls), delete all data, and export everything.
  Combined with no rate limiting and unbounded num_personas,
  this is a direct cost-amplification attack vector.
  3. No tests for scraper.py (358 lines) — The most complex
  module (retry logic, rate limiting, deduplication, region
  mapping) has zero test coverage.
  4. No SQL injection regression tests — While parameterized
  queries are used correctly, the dynamic WHERE clause
  construction in sessions.py has no tests verifying safety
  against malicious input.

  ---
  HIGH Priority Findings

  Security:
  - Wildcard CORS (allow_origins=["*"] +
  allow_credentials=True) — contradictory and dangerous
  - No rate limiting — Claude-calling endpoints can be spammed
  indefinitely
  - No CSRF protection in frontend
  - Exception details leaked to clients via
  HTTPException(detail=f"...{e}")

  Code Quality:
  - No connection pooling — new TCP connection per request via
  get_conn()
  - Unbounded recursion in fetch_json on 429 responses (never
  increments attempt)
  - insert_tags raises unhandled KeyError on unknown
  dimension/value pairs
  - O(n*m) PSM curve computation — should use numpy or
  pre-sorted binary search
  - Sequential Claude API calls — 50 personas = 50 serial
  round-trips

  Test Gaps:
  - No tests for load_and_tag.py, remove_megathreads.py,
  personas/selection.py
  - 3 API endpoints completely untested (DELETE, restore,
  permanent delete)
  - No input validation tests (zero/negative num_personas,
  empty question, invalid sector)
  - No export security tests (CSV injection, special
  characters)

  ---
  MEDIUM Priority Findings

  Security: num_personas has no upper bound, question has no
  max length, ILIKE wildcards not escaped, Content-Disposition
  header injection, no CSP headers, no npm audit in CI

  Code Quality: Duplicated _load_wtp_template across two files,
   purge_expired_sessions called on every GET, repeated
  conn/try/finally pattern (should be a FastAPI dependency),
  hardcoded Claude model/token limits, missing DB indexes on
  posts.author and metadata->>'sector', sequential API calls,
  ORDER BY RANDOM() on large tables, matplotlib missing from
  pyproject.toml dependencies

  Tests: Over-mocking in test_api.py (14 patches), fragile SQL
  string assertions in test_sessions.py, missing edge cases in
  tagger/embeddings/WTP tests, module-scoped DB fixtures
  accumulate data

  ---
  Recommended Remediation Order

  1. Add authentication — API key middleware or OAuth,
  especially on Claude-calling endpoints
  2. Add rate limiting — slowapi or similar, cap requests per
  IP/key
  3. Fix connection leaks — wrap all get_conn() calls in
  try/finally, or better: create a FastAPI dependency (yield
  conn)
  4. Add connection pooling —
  psycopg2.pool.ThreadedConnectionPool
  5. Bound inputs — num_personas: int = Field(ge=1, le=50),
  question: str = Field(max_length=2000)
  6. Restrict CORS — set allow_origins to your actual frontend
  domain
  7. Sanitize error responses — generic messages to client, log
   details server-side
  8. Add missing DB indexes — posts.author,
  posts.metadata->>'sector'
  9. Add scraper tests — mock requests.Session, test
  retry/filtering logic
  10. Add security regression tests — SQL injection, CSV
  injection, input validation boundaries
